# Browser Gateway extension — known issues & fixes — COMPLETED

**Status: COMPLETE (2026-06-02).** All 10 issues below are implemented and
verified (typecheck + lint + regression tests green). The one outstanding caveat
is a real-Chrome smoke run — see "Not yet live-tested" at the end.

Diagnosed 2026-06-02 from a live session driving Apple's App Store Connect /
Developer portal. Permissions in `manifest.json` are sufficient (`debugger`,
`scripting`, `tabs`, `downloads`, `<all_urls>`). All problems were in how
`background.js` used them. Ordered by impact.

### How #2 / #9 / #10 work now

- **All-frames actions (#2):** `runInTargetTab` injects into every frame
  (`allFrames: true`). Each frame returns a not-found sentinel when it lacks the
  element; `mergeFrameResults` picks the frame that matched. Selectors generated
  by `query_elements` in a child frame therefore resolve when `click`/`type`/
  `select` re-run across all frames. `wait_for` polls all frames (rather than a
  per-frame observer) so frames lacking the selector don't hold the call open
  until timeout.
- **Closed shadow roots (#2):** `open-shadow-roots.js` runs in the page's MAIN
  world at `document_start` and coerces `attachShadow({mode:'closed'})` to
  `open`, so the traversal can see component internals (the same technique
  Playwright/Puppeteer use). **Tradeoff:** this injects into every http(s) page,
  always, and makes all shadow roots externally visible. It only changes
  *visibility* (page code keeps its own root reference), but it is a global
  behaviour change for the user's browser — disable by removing the
  `content_scripts` entry in `manifest.json` if undesired. Requires Chrome 111+
  (`world: "MAIN"` in a manifest content script).
- **Custom selects (#9):** native `<select>` matches an option by value, label,
  then visible text; non-native dropdowns are opened with a real click and the
  option matching the requested text is clicked (short poll for async-rendered
  options).
- **Effect verification (#10):** a selector that matches nothing in any frame
  now throws, so the gateway records `failed` instead of a misleading success.
  `type`/`fill_form` return `valueAfter` + `valueApplied` as evidence. Deeper
  post-action diffing is still best done by a follow-up `snapshot`.

## What changed (resolution detail)

The implementation lives in `background.js` (the extension), `open-shadow-roots.js`
(new), `manifest.json`, and on the gateway side in
`src/main/browser-gateway/browser-existing-tab-operations.ts`. Regression
assertions are in `src/main/browser-gateway/browser-extension-assets.spec.ts`.
The original diagnosis (with the symptoms observed live) is preserved in git
history; line numbers below are intentionally omitted because they drift.

- **#1 / #6 / #7 Screenshots.** `captureVisibleTab` (focus-dependent, full-res
  PNG) was replaced by `captureTabScreenshot`, which uses CDP
  `Page.captureScreenshot` via the existing `withDebugger` helper. Captures a
  specific tab regardless of focus/visibility; defaults to JPEG q60 downscaled to
  ≤1280px via `Page.getLayoutMetrics` + `clip.scale`, and honors
  `maxWidth/maxHeight/fullPage` (now plumbed through the gateway screenshot
  command payload). The forced `focusTab` was removed.
- **#2 Frames + shadow roots.** `capturePageText` and `runInTargetTab` now inject
  with `allFrames: true`. Per-frame results are merged by `mergeFrameResults`,
  which picks the frame that matched, so selectors generated by `query_elements`
  inside an `<iframe>` resolve when actions re-run across frames. `wait_for`
  polls all frames (`waitForSelectorAcrossFrames`) instead of a per-frame
  observer. Closed shadow roots are made traversable by `open-shadow-roots.js`
  (MAIN world, `document_start`) — see the tradeoff note above.
- **#3 Clicks.** `dispatchRealClick` fires a full
  `pointerover → pointerdown → mousedown → pointerup → mouseup → click` sequence;
  synthetic `element.click()` is gone (it would double-fire default actions).
- **#4 query_elements.** Already exposed end-to-end as `browser.query_elements`
  (MCP tool + gateway routing + extension handler). No change needed.
- **#5 Typing.** `setNativeValue` calls the prototype `value` setter
  (`HTMLInputElement`/`HTMLTextAreaElement`/`HTMLSelectElement`) before
  dispatching `input`/`change`, so React/controlled inputs register the value.
- **#8 Bridge reliability.** Non-poll messages are buffered (`enqueueOutbox` /
  `flushOutbox`, capped) and flushed on reconnect; reconnect uses exponential
  backoff with jitter (`scheduleReconnect`), reset by healthy inbound traffic.
  Poll requests are never replayed.
- **#9 Selects.** `selectValue` matches native `<select>` options by
  value/label/text; non-native dropdowns are opened with a real click and the
  option matching the requested text is clicked (`findOptionByText`, short poll
  for async-rendered options).
- **#10 Effect verification.** A selector matching nothing in any frame now
  throws (`No element matches selector: …`), so the gateway records `failed`;
  `type`/`fill_form` return `valueAfter`/`valueApplied` as evidence.

## Not yet live-tested

These changes are verified by typecheck, lint, and substring regression tests,
but the extension itself has not been exercised in a real Chrome session. Highest
-value smoke tests after loading the unpacked extension: screenshot while Chrome
is **not** focused; click a React/portal control; snapshot a page with iframe
content; type into a React-controlled input; select from a custom dropdown.
