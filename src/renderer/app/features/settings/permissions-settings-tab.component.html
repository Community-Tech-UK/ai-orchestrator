Browser automation — things to know

When an agent controls a real web browser it can read pages, click buttons, and make network requests — so it needs broader access than a typical read-only task. The tips below help you stay in control.

Use a separate browser profile Keep the agent's browser sessions separate from your personal Chrome profile so your saved passwords and cookies stay private.
Start with session-level approval Approve browser access for this session only first. Once you're happy it's working as expected, you can make it permanent.
Only allow browser tools you recognise A browser tool can navigate, click, and read page content. Only approve tools from sources you trust.
