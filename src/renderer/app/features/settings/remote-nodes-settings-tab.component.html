Remote Nodes

Let another computer on your network do work for this app — running a web browser, using its GPU, or adding extra AI CLI capacity.

A remote node is another computer (for example your Windows PC) that does heavy jobs on this app's behalf — browser automation, GPU work, or extra AI CLI capacity. This machine stays in charge and simply hands out the work; the other computer runs it and reports back.

Server Configuration

How this machine listens for remote computers. The defaults work for most setups — only change these if a port is already taken or you run more than one separate group of machines.

The network port remote computers connect to. Default is 4878 — change it only if another program is already using that port.

Which connections to accept. 0.0.0.0 = any computer on your network (the normal choice). 127.0.0.1 = this machine only, so nothing else can connect.

A label that groups machines together. A remote computer only auto-finds this app when its namespace matches. Leave it as default unless you deliberately run several separate clusters.

Scrambles the traffic between this app and remote computers so others on the network can't read it. Recommended on shared or public Wi-Fi; optional on a trusted home network.
